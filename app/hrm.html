Hierarchical Reasoning Model (HRM)

Early 2025 (Sapient) | Hierarchical recursive

A multi-timescale recurrent transformer.

Overview & Core Idea

HRM consists of two recurrent modules: a low-level network that updates frequently and a high-level network that updates more slowly. They alternate in a single forward pass without explicit intermediate supervision.

[ Architecture Diagram Placeholder ]

Key Innovation vs Standard Transformers

Brain-inspired hierarchical recursion. Instead of static layers, HRM interleaves loops at two frequencies. It achieves deep “planning” using recursion rather than stacking layers.

Memory System & Compute

Both networks carry separate latents (fast and slow features), which are fed into each other.

Strengths

Extremely efficient learning: with only 1k training examples, HRM solves complex tasks that stump LLMs. Nearly perfect accuracy on hard Sudoku and mazes without CoT or pretraining.

Weaknesses

Task-specific (puzzles, symbolic tasks). It’s not a drop-in LLM replacement; no open-ended text generation ability. Requires tasks that fit its supervised setup.

Resources & Open Source

Official GitHub (sapientinc/HRM) and ArXiv paper. Foundation for TRM.
