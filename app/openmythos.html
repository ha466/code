OpenMythos

2026 (kyegomez/OpenMythos) | Strongly Recursive / Looped

A Recurrent-Depth Transformer implementing the rumored “Claude Mythos” design.

Overview & Core Idea

It splits the model into a Prelude, a looped Recurrent Block of shared layers, and a Coda. In one forward pass, it repeatedly applies the same small set of layers (weight-tied) up to max_loop_iters, effectively simulating a very deep network. Hidden state is updated each loop.

Key Innovation vs Standard Transformers

Parameter sharing via loop-depth. Instead of stacking e.g. 96 distinct layers, OpenMythos might loop 16 layers 6 times. Optional adaptive halting (ACT) can stop loops early. The feed-forward uses a fine-grained MoE (mixture-of-experts).

Memory System & Compute

All reasoning happens within this loop (there is no output until the end); hidden state is updated iteratively.

Strengths

Very parameter-efficient “deep thinking”: deeper effective depth with few weights. Adaptive compute (loop count can depend on input). Emergent subspace selectivity. Suspected to mirror state-of-the-art models (e.g. Claude).

Weaknesses

Largely theoretical/speculative; training stability and convergence can be tricky. Requires managing loop indices and potential hidden-state explosion. No published benchmarks (community analysis only).

Resources & Open Source

Open-source code on GitHub (kyegomez/OpenMythos). Lightweight enough to run locally (e.g. on a GPU laptop). Well-suited for custom iterative agents.
