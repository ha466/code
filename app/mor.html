Mixture-of-Recursions (MoR)

2025 (Google DeepMind + KAIST + Mila) | Hybrid Recursive + MoE Routing

Dynamic/adaptive recursion depth per token using lightweight routers.

Overview & Core Idea

Recursive Transformer combined with lightweight routers that dynamically assign per-token recursion depths. It shares parameters and employs selective KV caching.

[ Architecture Diagram Placeholder ]

Key Difference from Standard Transformers

Dynamic/adaptive recursion depth per token + parameter sharing. It merges recursion efficiency with MoE-style compute routing.

Memory System & Compute

Focuses attention only on "active" tokens, bypassing recursion for simple tokens to preserve compute and memory bandwidth.

Strengths & Real World Use Cases

Combines recursion efficiency with MoE-style routing for compute. Highly efficient for varied-complexity inputs.

  • Mixed-complexity document analysis.
  • Efficient inference on edge devices built on Llama backbones.
