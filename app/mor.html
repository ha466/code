Mixture-of-Recursions (MoR)

2025 (Google DeepMind et al.) | Strongly recursive (looped) with MoE routing

A recursive transformer that combines parameter sharing with token-level adaptive depth.

Overview & Core Idea

It uses a single shared stack of transformer layers across recursions, while lightweight routers dynamically decide how many loops each token needs. “Easy” tokens exit early; “hard” tokens recurse more. Only active tokens are attended to.

Key Innovation vs Standard Transformers

Integrates weight tying (shared layers) and Mixture-of-Experts-like routing. It addresses the “missing KV cache” and batching problems of naive early exit by sharing loops.

Memory System & Compute

A novel recursion-wise KV caching stores only active tokens’ keys/values, preserving memory bandwidth.

Strengths

MoR reaches a new efficiency frontier: for equal FLOPs/Model size it yields lower perplexity and higher accuracy than vanilla Transformers, with ~2× inference throughput advantage. Naturally allocates compute where needed.

Weaknesses

More complex training (must learn routers) and inference logic. Routers add overhead. Tuning the recursion policies can be tricky.

Resources & Open Source

Code on GitHub (raymin0223/mixture_of_recursions) built on Llama. Pretrained on FineWeb data. Good as a hybrid approach.
